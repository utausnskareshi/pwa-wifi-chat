# WiFi Chat

同一WiFiネットワーク上のiPhone同士でP2P（ピアツーピア）チャットができるPWA（Progressive Web App）です。

QRコードを使って相手を認識し、テキストメッセージおよびカメラロールの画像を送受信できます。

---

## スクリーンショット

| ホーム画面 | QRコード表示 | チャット画面 |
|:-:|:-:|:-:|
| アプリ起動時の画面 | ホスト側がQRを提示 | 接続後のチャット |

---

## 特徴

- **サーバー不要のP2P通信** — WebRTC DataChannel（PeerJS）を使用。メッセージはデバイス間を直接やり取りします
- **QRコードで簡単ペアリング** — 一方がQRコードを表示し、もう一方がカメラで読み取るだけで接続完了
- **2段階のQRスキャン方式** — カメラのリアルタイムスキャンが使えない場合（iOS PWAスタンドアロンモード等）は、写真撮影によるフォールバック方式に自動切替。カメラ許可の設定手順も画面内にガイド表示
- **テキスト & 画像チャット** — カメラロールの写真をそのまま送受信可能。大きな画像は自動リサイズして送信
- **PWA対応** — ホーム画面に追加してネイティブアプリのように使えます。Service Workerによるオフラインキャッシュ対応
- **ダークモード完全対応** — iOS のシステム設定に合わせてUIが自動切替。チャット入力欄の文字色・プレースホルダーもダークモードで正しく表示
- **iPhone Safari 最適化** — ノッチ・ダイナミックアイランド・ホームバーのセーフエリアに対応

---

## 動作環境

| 項目 | 要件 |
|------|------|
| デバイス | iPhone（iOS 14.5以上推奨） |
| ネットワーク | 2台が同一WiFiネットワークに接続されていること |
| ブラウザ | Safari（iOS）/ Chrome |
| ホスティング | GitHub Pages（HTTPS必須） |

> **注意:** WebRTC と Service Worker は **HTTPS** 環境でのみ動作します。GitHub Pages は標準でHTTPSを提供するため、そのまま使用できます。

## 使い方

### 1. アプリを開く

GitHub Pages で公開したURLを、チャットしたい2台のiPhoneで開きます。

```
https://<ユーザー名>.github.io/<リポジトリ名>/
```

### 2. ホーム画面に追加（推奨）

Safari の共有ボタン → 「ホーム画面に追加」でPWAとしてインストールすると、フルスクリーンで使えます。

### 3. チャット開始

**iPhone A（QRコードを表示する側）**
1. アプリを開き「**QRコードを表示する**」をタップ
2. QRコードが表示されるまで待つ（数秒）

**iPhone B（QRコードを読み取る側）**
1. アプリを開き「**QRコードを読み取る**」をタップ
2. カメラの使用許可を求められた場合は「**許可**」をタップ
3. カメラ起動後、iPhone A に表示されたQRコードを画面内に入れる
4. 自動で読み取られ、チャット画面へ遷移

> **カメラが起動しない場合:** カメラの権限が拒否されている場合は、画面にiPhoneの設定手順が表示されます。設定を変更するか、「**QRコードを撮影して読み取る**」ボタンからネイティブカメラで撮影して読み取ることもできます。

### 4. メッセージを送る

- **テキスト**: 入力欄に文字を入力して送信ボタン（➤）または Enterキーを押す
- **画像**: 左下の画像ボタン（🖼）をタップしてカメラロールから写真を選ぶ
- **画像の拡大**: チャット内の画像をタップするとフルスクリーンで表示

## 技術仕様

### アーキテクチャ

```
iPhone A                    PeerJSシグナリングサーバー         iPhone B
   |                               |                              |
   |--- createPeer() -----------> |                              |
   |<-- Peer ID割り当て ---------- |                              |
   |                               |                              |
   |--- QRコード表示（Peer ID）     |                              |
   |                               |                              |
   |                               | <--- createPeer() ---------- |
   |                               | ---> Peer ID割り当て -------> |
   |                               |                              |
   |                               | <--- connect(PeerID_A) ----- |
   | <--- 接続要求 ---------------- |                              |
   |                               |                              |
   |========= WebRTC DataChannel (P2P 直接通信) ==================|
   |                               |                              |
   |--- テキスト/画像 ============================================>|
   |<========================================== テキスト/画像 ----|
```

接続確立後のメッセージ通信はすべてP2Pで行われ、シグナリングサーバーを経由しません。

### 通信プロトコル

メッセージはすべて JSON 形式で DataChannel を通じてやり取りします。

#### テキストメッセージ
```json
{
  "type": "text",
  "content": "こんにちは！",
  "time": "14:30"
}
```

#### 画像転送（開始通知）
```json
{
  "type": "image-start",
  "id": "img-1700000000000-ab3x",
  "totalChunks": 12,
  "mimeType": "image/jpeg",
  "time": "14:31"
}
```

#### 画像転送（チャンクデータ）
```json
{
  "type": "image-chunk",
  "id": "img-1700000000000-ab3x",
  "index": 0,
  "data": "<Base64エンコードされた16KBのデータ>"
}
```

画像は Base64 エンコード後に 16KB ずつ分割して送信されます。受信側は全チャンクが揃った時点で画像を表示します。

### 使用ライブラリ

| ライブラリ | バージョン | 用途 |
|------------|-----------|------|
| [PeerJS](https://peerjs.com/) | 1.5.4 | WebRTC P2P通信（シグナリング + DataChannel） |
| [QRCode.js](https://github.com/soldair/node-qrcode) | 1.5.x | QRコード生成（Canvas描画） |
| [html5-qrcode](https://github.com/mebjas/html5-qrcode) | 2.3.8 | QRコードスキャン（カメラ制御） |

すべてのライブラリは `lib/` フォルダにローカル保存されています。CDNへの外部依存はありません。

## GitHub Pages へのデプロイ

### 手順

```bash
# 1. リポジトリを初期化（まだの場合）
cd WifiChat
git init
git add -A
git commit -m "feat: WiFi Chat PWA 初版"

# 2. GitHubにリポジトリを作成してプッシュ
git remote add origin https://github.com/<ユーザー名>/<リポジトリ名>.git
git branch -M main
git push -u origin main
```

### GitHub Pages を有効化

1. GitHubのリポジトリページを開く
2. **Settings** → **Pages** を選択
3. **Source** を「Deploy from a branch」に設定
4. **Branch** を `main` / `(root)` に設定して **Save**
5. 数分後に `https://<ユーザー名>.github.io/<リポジトリ名>/` で公開される

## トラブルシューティング

### QRコードが表示されない
- インターネット接続を確認してください（PeerJSシグナリングサーバーへのアクセスが必要です）
- ページをリロードして再試行してください

### カメラが起動しない / 「カメラにアクセスできません」と表示される
- Safari の設定でカメラのアクセスを許可してください
  1. iPhone の **設定** アプリを開く
  2. **Safari** を選択
  3. **カメラ** をタップ
  4. **許可** に変更する
  5. アプリに戻り、ページを **再読み込み** する
- HTTPS でアクセスしているか確認してください（HTTPではカメラにアクセスできません）
- iOS PWA のスタンドアロンモード（ホーム画面から起動）では `getUserMedia` が制限される場合があります。その場合は画面に表示される「**QRコードを撮影して読み取る**」ボタンをご利用ください

### 接続できない
- 2台が**同じWiFiネットワーク**に接続されているか確認してください
- QRコードが期限切れの場合は、ホスト側でページをリロードしてQRコードを再生成してください
- ファイアウォールやプロキシがWebRTC通信をブロックしている場合があります（企業・学校のネットワークで発生しやすい）

### 画像が送れない
- 対応形式: JPEG / PNG / GIF / WebP など `image/*` 形式
- 非常に大きな画像（数十MB以上）は送信に時間がかかる場合があります
- アプリは自動的に最大1200pxにリサイズして送信量を削減します

## 開発・カスタマイズ

### ローカルでの動作確認

ローカルでテストする場合は HTTPS が必要です。Python の簡易サーバーでは動作しない機能（カメラ等）があります。
`mkcert` などを使ってローカル HTTPS 環境を構築することを推奨します。

```bash
# Python で簡易HTTPサーバーを起動（カメラ機能はHTTPSが必要）
python -m http.server 3000
# → http://localhost:3000 でアクセス（localhostはHTTPでもカメラOK）
```

### キャッシュのクリア方法

Service Worker のキャッシュが古くなった場合は `sw.js` の `CACHE_NAME` のバージョン番号を上げてください。

```javascript
// sw.js
const CACHE_NAME = 'wifichat-v2'; // v1 → v2 のようにバージョンを上げる
```

### カスタマイズポイント

| 項目 | ファイル | 変数/クラス |
|------|----------|------------|
| テーマカラーの変更 | `css/style.css` | `:root` の `--primary`, `--secondary` |
| 画像リサイズの最大サイズ変更 | `js/app.js` | `resizeImage(file, 1200)` の `1200` |
| チャンクサイズの変更 | `js/app.js` | `const CHUNK_SIZE = 16000` |
